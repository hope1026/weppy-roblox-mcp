# Changelog

> AIがRoblox Studioで実行したすべての変更をセッション単位で記録し、変更要約/タイムライン/Before & Afterを提供します。

![Changelog — セッション別変更カード一覧](../../assets/screenshots/dashboard/dashboard_changelog1.png)

## 概要

ChangelogはAIエージェントがStudioで実行したゲーム変更履歴を追跡するページです。変更はセッション単位でグループ化され、各セッションでどのような種類の変更が何件発生したかを一目で把握できます。

## セッションカード一覧

Changelogの最初の画面はセッション別のカード一覧を表示します。

### セッション状態

各セッションカードには状態が表示されます：

| 状態 | 意味 |
|------|------|
| **Active** | 現在進行中のセッション（リアルタイム更新） |
| **Completed** | 完了したセッション |
| **Bootstrap** | 初期同期セッション |

### セッション要約

各カードには該当セッションの変更件数がカテゴリ別に要約されます：

- Script — スクリプト作成/修正件数
- Instance — インスタンス作成/削除/移動件数
- Property — プロパティ変更件数
- Lighting — ライティング/環境設定件数
- Terrain — 地形変更件数
- Asset — アセット挿入件数

### フィルタータブ

上部のタブでセッションをフィルタリングできます：
- **All** — すべてのセッション
- **Active** — 進行中のセッションのみ
- **Completed** — 完了したセッションのみ

## セッション詳細ビュー

セッションカードをクリックすると詳細ビューに入ります。

![Changelog詳細ビュー — 変更要約、タイムライン、Before & After](../../assets/screenshots/dashboard/dashboard_changelog2.png)

### Change Summary

セッションの変更を6つのカテゴリカードで可視化します：

| カテゴリ | アイコン | 説明 |
|---------|--------|------|
| Scripts | スクリプト | スクリプト作成/修正件数 |
| Instances | インスタンス | インスタンス作成/削除/移動件数 |
| Properties | プロパティ | プロパティ変更件数 |
| Lighting | ライティング | ライティング/環境変更件数 |
| Terrain | 地形 | 地形変更件数 |
| Assets | アセット | アセット挿入件数 |

### Change Timeline

セッション内のすべての変更を時系列で一覧表示します。

- 各項目にタイムスタンプ、変更カテゴリタグ、対象パスが表示されます
- **Category**ドロップダウンで特定のカテゴリのみフィルタリングできます
- 項目をクリックするとBefore & After比較ビューが展開されます

### Before & After

変更前後のデータを比較します。変更タイプに応じて異なるレベルの情報を提供します：

| 信頼度 | 意味 | 例 |
|--------|------|------|
| **exact** | 変更前後の値が正確に記録されている | プロパティ変更、スクリプト修正 |
| **partial** | 一部の情報のみ記録されている | 複合変更 |
| **after-only** | 変更後の値のみ存在する | 新規インスタンス作成 |
| **intent-only** | 意図のみ記録されている | 削除など |

## 活用事例

### 作業検証

```
「AIが修正したスクリプトが何か確認したい」
```

ChangelogでActiveセッションのScriptカテゴリをフィルタリングすると、修正されたスクリプト一覧と変更前後のコードを比較できます。

### 変更追跡

```
「昨日のセッションでLightingをどう設定したか確認したい」
```

CompletedタブでそのセッションをLightingカテゴリでフィルタリングすると、変更履歴と設定値を確認できます。

### 問題デバッグ

```
「どの時点で特定のインスタンスが削除されたか探す必要がある」
```

タイムラインでInstanceカテゴリをフィルタリングし、deleteタイプの変更を時系列で追跡できます。
